# wdttgukji

**어떤 세계관이든 넣으면 코에이급 전략 게임이 나오는 에이전틱 게임 생성 파이프라인.**

첫 번째 테마: 삼국지.

---

## 왜 이걸 만드나

침착맨의 삼국지8 리마스터 부사인전, 삼국지14 적벽대전 망한 유비전을 보고 시작됨.
코에이가 30년 동안 쌓은 이벤트/캐릭터/시스템을, AI 파이프라인으로 100배 확장하면 어떨까?

핵심 가설:
- 장수 1000명의 "영혼"을 구조화하면 AI가 그 장수답게 행동할 수 있다
- 연의/정사/대체역사를 학습하면 AI가 기존 이벤트의 100배를 생성할 수 있다
- 게임 메카닉(내정/전투/외교/관계)은 테마에 독립적으로 추상화 가능하다

## 기술 스택

| 레이어 | 기술 | 이유 |
|---|---|---|
| 게임 엔진 | HTML Canvas + vanilla JS (→ WebGL/Three.js 후순위) | 프로토타입은 2D, 리마스터급 연출은 후순위로 확장 |
| 앱 래퍼 | Tauri (Rust) | 웹 기술 그대로 + 가벼운 exe/app 패키징 (~10MB) |
| AI 파이프라인 | LLM API (GPT-4o / Claude) | 이벤트 생성, 캐릭터 행동, 테스트 페르소나 |
| 데이터 | JSON + SQLite | 장수/이벤트는 JSON, 게임 상태는 SQLite (로컬) |
| 배포 | 로컬 실행 (exe/app) | 호스팅 비용 $0. AI API 호출만 인터넷 필요 |

## 데이터 파이프라인

### Phase 0: 데이터 수집 (현재 단계)

```
소스                          → 크롤링          → 가공
────────────────────────────────────────────────────────
나무위키 삼국지 인물            → characters.json → soul.md ×1000
나무위키 삼국지 연의/정사       → events.json     → event schema
코에이 삼국지 8/11/14 위키      → mechanics.json  → game design doc
유튜브 (침착맨 등) 제목/댓글   → sentiment.json  → 분기점 포인트
Reddit/커뮤니티               → community.json  → 유저 선호도
중국/일본 삼국지 DB            → intl.json       → 크로스 검증
```

### 수집 대상

**캐릭터 데이터 (~1000명)**
- 이름 (한국어, 한자, 영문)
- 능력치 (무력, 지력, 통솔, 정치, 매력)
- 전기 (출생~사망, 주요 사건)
- 소속 세력 변천사
- 인간관계 (의형제, 군신, 혼인, 원수)
- 성격/기질 → soul.md의 근거

**이벤트 데이터 (~500개 기존 + AI 확장)**
- 이벤트명
- 시기 (연도/시나리오)
- 참여 장수
- 장소
- 전개 + 분기 조건
- 결과 + 영향

**게임 메카닉 데이터 (코에이 분석)**
- 내정 시스템 (개간/상업/기술/치안)
- 전투 시스템 (병종/진형/일기토)
- 외교 시스템 (동맹/연합/계략)
- 관계 시스템 (친밀도/충성도/의형제)
- 이벤트 트리거 조건

## 코어 게임 메카닉 (코에이 분석 기반 추상화)

삼국지에 묶이지 않고, 어떤 테마에든 적용 가능한 추상 메카닉:

### 1. 캐릭터 시스템
```yaml
Character:
  identity: { name, title, faction }
  stats: { military, intellect, leadership, politics, charisma }
  soul: "성격/기질/가치관 — AI 행동의 근거"
  relationships: [{ target, type, intensity }]  # 의형제, 원수, 군신
  loyalty: number     # 세력 충성도
  ambition: number    # 독립/배신 확률에 영향
```

### 2. 이벤트 시스템
```yaml
Event:
  id: string
  trigger: { conditions: [...] }     # 시기, 인물 상태, 세력 관계 등
  narrative: string                  # AI 생성 or 수동 작성
  choices: [{ text, effects, next_event }]
  participants: [character_id]
  historical_basis: "연의 XX회" | "정사 XX전" | "AI 생성"
```

### 3. 세력/영토 시스템
```yaml
Territory:
  cities: [{ name, population, economy, defense, governor }]
  army: { soldiers, morale, supply }
  diplomacy: [{ target_faction, relation, treaties }]
```

### 4. 턴 구조
```
매 턴 (= 1개월):
  1. 이벤트 체크 → 트리거 조건 충족된 이벤트 발화
  2. 내정 페이즈 → 행동력 소비해서 도시 관리
  3. 외교 페이즈 → 동맹/계략/등용
  4. 전투 페이즈 → 출진/방어/일기토
  5. AI 행동 → 플레이어 외 세력 자동 행동
  6. 결산 → 자원/인구/충성도 갱신
```

## AI 생성 파이프라인 (기획팀)

### soul.md 생성
```
[장수 원본 데이터] → [LLM] → soul.md
                              ├── 성격 요약 (MBTI식 기질)
                              ├── 가치관 (충의/야망/가족/명예)
                              ├── 의사결정 패턴
                              ├── 대화 톤
                              └── 역사적 행동 근거
```

### 이벤트 100배 확장
```
[기존 이벤트 500개] → [LLM + 연의/정사 컨텍스트] → 50,000개 이벤트
                                                    ├── "만약 ~했다면" 대체역사
                                                    ├── 캐릭터 간 관계 이벤트
                                                    ├── 계절/날씨/재해 이벤트
                                                    └── 랜덤 조우/발견 이벤트
```

### AI 침착맨 테스트
```
[침착맨 영상 자막/댓글 데이터] → [LLM 페르소나] → AI 플레이어
                                                  ├── 플레이 스타일 학습
                                                  ├── 선택지 반응 예측
                                                  ├── "이거 개사기잖아" 밸런스 탐지
                                                  └── 재미 포인트 피드백
```

## 에이전트 시스템 상세 — 뭘로 뭘 하나

### 사용 도구/서비스

| 도구 | 용도 | 비용 |
|---|---|---|
| **GPT-4o-mini** | 이벤트 대량 생성, soul.md 생성, NPC 대사 | ~$0.15/1M input, $0.60/1M output |
| **GPT-4o / Claude** | 품질 검수, 테스트 페르소나, 복잡한 시나리오 | ~$2.50~$15/1M tokens |
| **Playwright** | 웹 크롤링 (나무위키, 위키, 커뮤니티) | 무료 (오픈소스) |
| **Node.js** | 크롤링/파이프라인/게임 서버 전체 | 무료 |
| **Vercel** | 프론트엔드 + API 호스팅 | 무료 (Hobby) ~ $20/월 (Pro) |
| **Railway** | AI 워커 (이벤트 생성, 테스트 루프) | ~$5/월 (가벼운 워커) |
| **Supabase** | 게임 데이터 저장, 유저 세이브 | 무료 (500MB) ~ $25/월 |

### 에이전트별 상세

**데이터수집 에이전트** — 비용: 거의 0
```
도구: Playwright + Node.js fetch
하는 일:
  1. 나무위키/위키백과 삼국지 인물 페이지 크롤링 → JSON
  2. 코에이 위키 게임 데이터 크롤링 → JSON
  3. 유튜브 제목/자막 추출 → JSON
  4. Reddit/디시/루리웹 센티먼트 수집 → JSON
  5. 중국(百度百科)/일본(ニコニコ大百科) 크롤링 → JSON
비용: 서버 비용만 (크롤링 자체는 무료)
```

**기획 에이전트 (AI 생성)** — 비용: 핵심 지출
```
도구: GPT-4o-mini (대량) + GPT-4o (검수)
하는 일:
  1. 장수 1000명 soul.md 생성
     - 입력: 크롤링된 전기/능력치/관계 데이터
     - 출력: 구조화된 성격/가치관/행동패턴 문서
     - 예상: 1000명 × ~500 토큰 = 약 $0.30 (4o-mini)
  2. 이벤트 50,000개 생성
     - 입력: 기존 500개 이벤트 + 세계관 컨텍스트
     - 출력: 트리거 조건 + 선택지 + 결과가 있는 이벤트
     - 예상: 50,000개 × ~1000 토큰 = 약 $30 (4o-mini)
  3. 품질 검수 (상위 10% 샘플링)
     - GPT-4o로 일관성/재미/밸런스 체크
     - 예상: 5,000개 × ~2000 토큰 = 약 $25 (4o)
```

**테스트 에이전트 (AI 플레이어)** — 비용: 월간 운영비
```
도구: GPT-4o (페르소나) + 게임 엔진 API
하는 일:
  1. AI 침착맨 페르소나로 게임 플레이
  2. 매 선택지에서 반응/피드백 생성
  3. 밸런스 이상 감지 → 리포트
  4. 100회 자동 플레이 → 통계 분석
예상: 월 $10~30 (테스트 빈도에 따라)
```

### 예상 비용 요약

| 항목 | 1회성 | 월간 |
|---|---|---|
| **데이터 크롤링** | $0 | $0 |
| **soul.md 생성 (1000명)** | ~$1 | - |
| **이벤트 생성 (50,000개)** | ~$30 | - |
| **품질 검수** | ~$25 | - |
| **AI 테스트 플레이** | - | ~$20 |
