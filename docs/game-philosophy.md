# Game Philosophy

`wdttgukji`는 “정보를 많이 보여주는 전략 대시보드”가 아니라, “몇 번의 결정을 무겁게 체감하게 하는 코에이식 역사 전략극”이어야 한다.

이 문서는 app-surface, map, command, copy, agent routing이 전부 같은 철학을 따르도록 고정하는 계약이다.

## 1. 플레이어 판타지

플레이어는 숫자를 정리하는 관리자나 운영 대시보드 사용자가 아니다.

플레이어는:
- 세력의 입장에서 전장을 읽고
- 이번 턴 가장 중요한 한두 결정을 내리고
- 그 결과가 다음 달 판세로 되돌아오는 것을 체감하는 사람이다

핵심은 “많은 행동”이 아니라 “무거운 판단”이다.

## 2. 우리가 만들고 싶은 감각

- 게임은 제품 설명처럼 읽히면 안 된다.
- 화면은 툴처럼 보이기보다 장면처럼 보여야 한다.
- 패널은 많아도 되지만, 플레이어의 시선은 항상 하나의 질문으로 모여야 한다.
- 플레이어는 `지금 무엇을 고르는가`를 먼저 이해해야지, `이 위젯이 어떤 상태인가`를 먼저 해석하면 안 된다.

한 줄로 줄이면:

> 한 화면은 하나의 극적 질문만 던져야 한다.

예시:
- 시작 화면: `누구의 깃발을 들 것인가`
- 세력 선택: `이 세력의 첫 세 턴은 어떤 성격인가`
- 전장 선택: `지금 어느 거점을 붙잡아야 하는가`
- 명령 장면: `이번 턴 무엇을 실행할 것인가`
- 턴 결산: `내 선택이 판세를 어떻게 바꿨는가`

## 3. 코에이식 철학에서 가져올 것

`wdttgukji`는 코에이의 복잡함을 복제하는 게 아니라, 코에이가 잘했던 리듬을 가져와야 한다.

- 전투는 준비의 결산이어야 한다.
- 맵은 배경이 아니라 전략 보드여야 한다.
- 세력 차이는 수치 차이보다 플레이 감각 차이로 읽혀야 한다.
- 같은 턴이라도 위/촉/오는 “왜 지금 이 행동을 해야 하는가”가 달라야 한다.
- 깊이는 정보량이 아니라, 선택의 맥락에서 나온다.

## 4. 화면 철학

### 4-1. 한 화면, 한 주연

각 장면에는 주연이 하나만 있어야 한다.

- 시작 화면의 주연: 세력 선택과 출정 CTA
- 전장의 주연: 맵
- 명령 장면의 주연: 카드 선택과 확정
- 결산 장면의 주연: 결과와 다음 압박

주연을 빼앗는 보조 패널은 접거나 숨긴다.

### 4-2. 맵 우선

- 맵은 항상 “전략 공간”처럼 보여야 한다.
- selection-focus에서도 맵은 남은 슬롯이 아니라 중심 무대여야 한다.
- 같은 설명을 맵 패널과 우측 rail이 중복해서 말하면 맵이 죽는다.
- 맵이 약해지면 텍스트를 더하는 게 아니라, 주변 내레이터를 줄인다.

### 4-3. 결정 우선

- command 화면은 설명 장면이 아니라 결정 장면이다.
- 카드, 즉시 효과, 확정 버튼이 first frame 주연이어야 한다.
- “확정선”, “검토 칸”, “실행면” 같은 메타 설명은 보조 badge 수준이면 충분하다.
- 헤드라인은 반드시 `무슨 조치인가 / 왜 지금인가 / 즉시 무엇이 바뀌는가`를 말해야 한다.

### 4-4. 문구 우선순위

좋은 문구:
- 전황
- 위험
- 선택
- 결과

나쁜 문구:
- UI 플로우 설명
- 시스템 자기소개
- 같은 doctrine 문장의 반복
- product/spec 문체

## 5. 삭제 기준

다음 중 하나면 삭제 또는 접기 대상이다.

- 플레이어가 지금 내릴 결정을 직접 돕지 않는다
- 같은 정보를 다른 패널이 이미 말하고 있다
- 세계관 언어가 아니라 제작자 언어로 쓰였다
- first frame에서 맵/카드/결정 버튼을 밀어낸다
- “멋있어 보이지만” 실제로는 대시보드 밀도만 높인다

## 6. 첫 10분 계약

첫 10분은 아래 루프로 읽혀야 한다.

1. 세력을 고른다
2. 첫 거점을 고른다
3. 명령 하나를 확정한다
4. 결과를 본다
5. 다음 턴 압박을 이해한다

이 루프를 흐리는 요소는 미뤄도 된다. 이 루프를 강화하는 요소가 최우선이다.

## 7. Agent Routing

이 철학을 실제 작업에 연결할 때 기본 셀은 아래다.

- lead: `koei-systems-designer`
- fit/scene: `ux-stage-director`
- map presence: `map-art-director`
- implementation: `engine-integrator`
- copy clean-up: `content-planner` (secondary)
- weirdness/playfeel check: `qa-persona-simulator` (late pass only)

중요:
- `content-planner`는 메인 철학 리드가 아니다
- `qa-persona-simulator`는 철학을 세운 뒤 검증하는 역할이지, 철학을 대신 만들지 않는다

## 8. Review Questions

UI나 장면을 바꾸기 전에 반드시 묻는다.

- 이 화면의 극적 질문은 하나인가
- 주연은 무엇인가
- 맵이 주연이어야 하는데 보조 패널이 더 크게 말하고 있지 않은가
- 플레이어가 지금 내릴 결정을 한 문장으로 말할 수 있는가
- 이 문장이 게임 안 사람의 언어인가, 제작자 문서의 언어인가
- 이 요소가 없으면 게임이 약해지는가, 아니면 더 선명해지는가
