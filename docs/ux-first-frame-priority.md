# UX First-Frame Priority

`ux-stage-director` 관점에서 정리한 첫 프레임 우선순위 메모.

기준 viewport: `1512x982`

## 시작 화면

첫 프레임에서 반드시 보여야 하는 것:
- 게임 제목과 톤
- 메인 CTA
- 최근 저장
- 시나리오 개요

첫 프레임에서 줄이거나 제거할 것:
- 장식 지도 잔상
- 장식성 문장 반복
- 동일 정보를 반복하는 카드

## 세력 선택

첫 프레임에서 반드시 보여야 하는 것:
- 좌측 세력 리스트
- 우측 선택 세력 도감
- 중앙 전장 프리뷰

줄이거나 제거할 것:
- 장식형 비교 카피
- 긴 보조 설명

## 인트로

첫 프레임에서 반드시 보여야 하는 것:
- 세력 이름
- 첫 3턴 가이드
- 시작 버튼

줄이거나 제거할 것:
- 긴 서사 본문
- 반복되는 수치 카드

## 전장 첫 프레임

첫 프레임에서 반드시 보여야 하는 것:
- 상단 HUD
- 전장 맵
- 짧은 브리프
- 기본 행동 진입점

첫 프레임에서 접을 것:
1. `war-room` 3칸 카드 그리드
2. 하단 선택 패널 보조 카드
3. 현장 반응 본문
4. 결과 배너 보조 문장

## 운영 원칙

- 첫 프레임은 “정보 최대화”가 아니라 “주의 흐름 고정”이다.
- 패널은 많아도 되지만 첫 진입에서 모두 펼치면 안 된다.
- MacBook 14에서 사용자가 전체 페이지를 스크롤하려고 하는 순간 UX 설계 실패로 본다.
