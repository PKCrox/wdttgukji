# MacBook 14 UX Contract

`wdttgukji`의 기본 플레이 기준 화면은 `MacBook Pro 14"`의 브라우저 viewport인 `1512x982`다.

이 문서는 레이아웃 핫픽스를 막고, QA와 UI 작업의 기준을 고정하기 위한 계약이다.

## 1. 기준 뷰포트

- Primary: `1512x982`
- Secondary: `1280x800`
- Large Desktop Reference: `2560x1440`

원칙:
- 모바일 반응형을 우선순위에 두지 않는다.
- 큰 화면은 정보 밀도를 늘리기보다 여백만 늘린다.
- 레이아웃 승인은 반드시 `1512x982` 기준으로 먼저 통과해야 한다.

## 2. 첫 10분 화면 규칙

### 시작 화면
- 스크롤 없이 첫 화면 전체가 보여야 한다.
- 장식 지도/잔상은 플레이 장면을 방해하지 않아야 한다.
- CTA, 최근 저장, 시나리오 개요가 모두 첫 프레임 안에 있어야 한다.

### 세력 선택
- 좌측 세력 리스트, 중앙 전장 프리뷰, 우측 세력 도감이 한 화면에 보여야 한다.
- 스크롤은 각 패널 내부에서만 허용한다.
- 전체 페이지가 세로로 밀려서는 안 된다.

### 인트로
- 출정 정보와 시작 버튼이 첫 프레임 안에 들어와야 한다.
- 서사 텍스트가 길면 본문 패널 내부 스크롤로 처리한다.

### 전장 첫 프레임
- 상단 HUD, 전장 맵, 핵심 브리프, 기본 선택 패널이 한 화면에 들어와야 한다.
- 브리프는 `읽는 카드`이지 `스크롤해야 보이는 박스`가 아니어야 한다.
- 도시를 선택하지 않아도 전장 중심 화면이 보여야 한다.
- 첫 프레임에서 전장 허브가 잘리지 않아야 한다.

## 3. 레이아웃 규칙

- 프레임은 viewport에 꽉 차지 않는다. 상하 여백은 항상 남긴다.
- 좌우 여백은 너무 크지 않아야 한다. 맥북14 기준 좌우 잔여 공간은 “장식 여백” 수준만 허용한다.
- 전장 화면에서 높이를 많이 먹는 요소는 순서대로 접는다.
  1. `war-room` 카드 그리드
  2. 하단 선택 패널 보조 정보
  3. 결과 배너 높이
  4. 턴 개시 카드 높이

## 4. 금지 사항

- CSS 미디어쿼리만 계속 덧대는 식의 핫픽스
- 기준 viewport 없이 “눈대중”으로 조정
- 시작 전 화면에 전장용 지도 질감/잔상을 남기는 것
- 캔버스 프로토타입 지도를 최종 아트처럼 취급하는 것

## 5. 지도 파이프라인 계약

- 지도는 `베이스맵 아트`와 `상호작용 오버레이`로 분리한다.
- 베이스맵은 SVG 또는 이미지 자산이 담당한다.
- 캔버스는 도시 선택, 전선, 이동, 펄스 같은 인터랙션만 담당한다.
- “맵 퀄리티 개선”을 CSS나 stroke 튜닝으로 해결하려고 하지 않는다.

## 6. 승인 기준

다음 항목이 모두 충족되어야 레이아웃 승인이다.

- `1512x982`에서 시작 화면이 스크롤 없이 완전 노출
- `1512x982`에서 전장 첫 프레임이 스크롤 없이 완전 노출
- 전장 프레임이 viewport 밖으로 삐져나가지 않음
- 시작 전 화면에서 지저분한 지도 잔상 없음
- 지도는 “개발용 다이어그램”이 아니라 게임용 전략 지도 구조로 보임
