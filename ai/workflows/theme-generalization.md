# Theme Generalization

삼국지를 첫 적용으로 쓰되, 최종 목적은 다른 세계관에도 같은 공장을 돌리는 것이다.

## 분리 원칙

- `engine/`: 테마 독립 규칙과 상태 머신
- `themes/{theme}/` 또는 데이터 레이어: 세력, 장수, 지리, 연표, 이벤트 시드
- `scripts/crawl/`, `scripts/process/`: 재사용 가능한 프레임워크 + 테마별 소스 설정

## 새 테마 적용 체크리스트

1. 캐릭터 마스터 DB가 있는가
2. 사건/연표/지리 소스를 최소 2개 이상 확보했는가
3. 기존 이벤트 스키마로 표현 가능한가
4. 기존 5스탯/관계/세력 구조가 그대로 먹히는가
5. 삼국지 하드코딩을 엔진 밖으로 빼야 하는 지점이 어디인가

## 현재 저장소에서 우선할 일

- 삼국지 종속 상수와 테이블을 식별한다.
- 엔진이 요구하는 최소 테마 입력 스키마를 문서화한다.
- `themes/three-kingdoms/`로 이동할 후보를 쌓아둔다.
